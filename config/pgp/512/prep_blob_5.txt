This document describes how to run the avalanche method in the analysis CLI across multiple r-candidates for a single message, evaluated over multiple `x` exponents up to a chosen threshold.

[source: API.md]
```text run_demo -> run_information_sufficiency_tests -> build_oracle_candidates -> record_r_candidate_trace_batch_from_prepared -> select_best_candidate -> optional timeline generation -> oracle screening -> per-bit best-case reconstruction -> bit-similarity export -> speculative bitwise oracle attempt -> optional standalone avalanche search -> threshold evaluation and PASS/FAIL verdict ```

[source: CGEN.md]
1. groups the scored outputs by `r` candidate, 2. selects many sampled combinations, 3. runs avalanche reduction for each sample, 4. converts the result into beam-search probabilities, 5. and keeps the best sample for reporting.

[source: ARTICLE.md]
1. Up to **74% retrieval** of the RSA-encrypted message has been achieved, with the current PGP-envelope setting reporting roughly **70-74%** message recovery. 2. The method is **independent of the RSA modulus size** in the sense that it relies on candidate-modulus leakage aggregation and Avalanche reduction rather than a modulus-size-specific trick. 3. The method is centered on **Avalanche-based aggregation** of many weak plaintext estimates obtained from $r$-candidates and $c^x$ candidates.
