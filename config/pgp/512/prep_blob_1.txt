[source: CONFIGS.md]
# Primary configs - `config/rsa_config_small_batch.json`: Main small-batch Avalanche config used by `make demo` and the current `analysis.rs` proof of concept. - `config/rsa_config_small_public_key_test.json`: Small public-key regression config that exercises public-key analysis plus a private-key verification peek. - `config/rsa_config_small_public_key_demo_<modulus-bits>.json`: Generated by `scripts/run_small_public_key_demo.sh` from the small-batch config and wired to the first extracted PKESK ciphertext under `config/pgp/<modulus-bits>/`. - `config/rsa_config.json`: Baseline general-purpose config with the broader engine schema documented below.

[source: DEMO.md]
**Public-key encrypted-file demo** - `scripts/run_small_public_key_demo.sh` calls `scripts/do_prep_pgp.sh`, defaults to `MODULUS_BITS=512`, generates armored OpenPGP messages under `config/pgp/<modulus-bits>/`, extracts the first PKESK ciphertext into a generated small-batch-derived config, and then runs `scripts/run_small_batch_beam.sh` against that encrypted fixed-file input. - It also stores `prep_blob_1.pkcs1_v1_5_block.hex` and `prep_blob_1.pkcs1_v1_5_payload.hex` beside `prep_blob_1.asc`, then fails unless the final `Avalanche global majority vote` hex equals the stored payload hex. ```bash RUNS=1 scripts/run_small_public_key_demo.sh ```
